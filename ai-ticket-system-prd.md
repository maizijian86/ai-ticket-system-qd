## 1. 项目概述

### 1.1 项目背景

随着企业规模的扩大和业务复杂度的增加，传统工单管理系统面临以下挑战：

- **工单分类效率低**：人工分类耗时长、易出错
- **处理人分配不均**：依赖人工分配，导致部分人员负载过重
- **知识复用困难**：历史解决方案难以快速检索利用
- **响应周期长**：新员工处理新问题时缺乏有效指导

### 1.2 项目目标

构建一套 **AI 驱动的智能工单管理系统**，通过 AI 技术实现工单处理全流程的智能化升级，提升企业运维效率 40% 以上。

### 1.3 核心价值

| 价值点 | 描述 | 预期收益 |
|--------|------|----------|
| 智能分类 | AI 自动识别工单类型，准确率 > 90% | 减少 80% 分类时间 |
| 精准派单 | 基于技能画像智能匹配处理人 | 处理效率提升 30% |
| RAG 问答 | 检索历史方案 + AI 生成回复 | 首次解决率提升 25% |
| 自动摘要 | 长对话自动生成摘要 | 阅读时间减少 50% |

### 3.2 功能详细描述

#### F1: 工单管理

**F1.1 工单创建**
- 支持文本输入 (必填)
- 支持图片上传 (可选，最多 5 张)
- 支持附件上传 (可选，限 PDF/Word/Excel，单个 < 10MB)
- 支持选择紧急程度
- 系统自动进行 AI 分类和优先级评估

**F1.2 工单查看**
- 工单列表：支持分页、筛选 (状态/分类/优先级/时间)、搜索
- 工单详情：完整内容、对话 Timeline、处理记录
- 工单导出：支持 Excel 导出

#### F2: AI 智能分类

| 类别 | 说明 | 示例 |
|------|------|------|
| Bug | 系统故障、功能异常 | "登录按钮点击无反应" |
| 咨询 | 问题咨询、功能询问 | "如何修改个人信息?" |
| 投诉 | 用户不满、投诉反馈 | "服务态度太差" |
| 建议 | 改进建议、新需求 | "希望增加夜间模式" |
| 其他 | 不属于以上类别 | - |

**分类流程：**
```
用户提交工单 → AI 文本分类 → 置信度计算 
    → 置信度 > 85%: 自动分类
    → 置信度 60-85%: 建议分类 + 人工确认
    → 置信度 < 60%: 标记为"待分类"
```

#### F3: AI 优先级评估

| 优先级 | 定义 | 触发条件 |
|--------|------|----------|
| P0 - 紧急 | 核心功能不可用 | 包含"崩溃"、"无法使用"、"紧急"等关键词 |
| P1 - 高 | 影响主要流程 | 非核心功能故障、影响多人 |
| P2 - 中 | 影响部分功能 | 单用户问题、功能瑕疵 |
| P3 - 低 | 轻微问题 | 咨询、建议、轻微体验问题 |

**评估因素：**
- 文本关键词匹配 (权重 30%)
- 历史相似工单优先级分布 (权重 25%)
- 用户等级 (权重 15%)
- 重复提交检测 (权重 20%)
- 附件类型判断 (权重 10%)

#### F4: AI 推荐处理人

**推荐算法：**
```
推荐分数 = 技能匹配度 × 0.4 
         + 历史解决率 × 0.25 
         + 用户满意度 × 0.2 
         + (1 - 当前负载率) × 0.15
```

**技能画像示例：**
```json
{
  "user_id": "u123",
  "skills": ["java", "mysql", "redis", "kafka"],
  "expertise_level": {
    "java": 5,
    "mysql": 4,
    "redis": 3
  },
  "metrics": {
    "resolved_count": 150,
    "avg_resolution_hours": 2.5,
    "satisfaction_score": 4.8
  }
}
```

#### F5: RAG 智能回复

**RAG 流程：**
```
用户问题 → 向量检索 (pgvector) 
        → Tavily 外部搜索 
        → 结果融合排序 
        → LLM 生成回复建议
```

**回复建议展示：**
```
┌─────────────────────────────────────────┐
│ 💡 AI 建议回复                           │
├─────────────────────────────────────────┤
│ 参考来源:                                │
│  1. 历史工单 #1234 (相似度 92%)          │
│  2. 知识库 KB-567 (相似度 88%)          │
│  3. 官方文档 (Tavily)                   │
├─────────────────────────────────────────┤
│ 建议回复:                               │
│ "您好，关于登录问题，请尝试以下步骤：..." │
├─────────────────────────────────────────┤
│ [ ] 一键采纳    [ ] 人工修改后发送        │
└─────────────────────────────────────────┘
```

#### F6: AI 工单摘要

**摘要内容：**
- 问题概述 (1-2 句话)
- 关键信息：涉及系统、操作步骤、错误信息
- 处理过程：处理人、耗时、解决方案
- 结论：是否解决、用户满意度

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 2s |
| API 响应时间 | < 500ms (普通接口), < 3s (AI 接口) |

- 系统可用率 > 99.5%
- 核心功能有备用方案 (AI 失败时人工处理)
- 数据定期备份 (每日全量 + 增量)

- 敏感数据加密存储

- 操作日志审计

- 支持多租户
- AI 模型可替换 (LangChain4j 支持多 provider)
- 知识库可导入导出

## 6. 数据库设计

### 6.1 核心表结构

#### 6.1.1 工单表 (ticket)

```sql
CREATE TABLE ticket (
    id              BIGSERIAL PRIMARY KEY,
    title           VARCHAR(500) NOT NULL,
    content         TEXT NOT NULL,
    category        VARCHAR(50),           -- AI 分类结果
    category_confidence DECIMAL(5,2),     -- 分类置信度
    priority        VARCHAR(10) DEFAULT 'P2',
    priority_score  DECIMAL(5,2),          -- AI 优先级评分
    status          VARCHAR(20) DEFAULT 'open',
    
    -- 提交者
    creator_id      BIGINT NOT NULL,
    creator_name    VARCHAR(100),
    
    -- 处理人
    handler_id      BIGINT,
    handler_name    VARCHAR(100),
    
    -- AI 推荐
    recommended_handler_id BIGINT,
    recommend_reason TEXT,
    
    -- 摘要
    ai_summary      TEXT,
    
    -- 附件
    attachments     JSONB,
    
    -- 时间
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    resolved_at     TIMESTAMP,
    closed_at       TIMESTAMP,
    
    -- 软删除
    deleted_at      TIMESTAMP,
    
    INDEX idx_category (category),
    INDEX idx_priority (priority),
    INDEX idx_status (status),
    INDEX idx_creator (creator_id),
    INDEX idx_handler (handler_id),
    INDEX idx_created (created_at)
);
```

#### 6.1.2 工单评论表 (ticket_comment)

```sql
CREATE TABLE ticket_comment (
    id              BIGSERIAL PRIMARY KEY,
    ticket_id       BIGINT NOT NULL,
    user_id         BIGINT NOT NULL,
    user_name       VARCHAR(100),
    content         TEXT NOT NULL,
    is_internal     BOOLEAN DEFAULT FALSE,  -- 是否内部留言
    is_ai_suggested BOOLEAN DEFAULT FALSE,  -- 是否 AI 建议
    
    -- AI 回复参考
    references      JSONB,  -- [{"source": "历史工单", "similarity": 0.92}]
    
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (ticket_id) REFERENCES ticket(id)
);
```

#### 6.1.3 用户技能画像表 (user_skill)

```sql
CREATE TABLE user_skill (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT UNIQUE NOT NULL,
    user_name       VARCHAR(100),
    
    -- 技能标签
    skill_tags      TEXT[],  -- ['java', 'mysql', 'redis']
    
    -- 技能等级 (1-5)
    expertise_level JSONB,  -- {"java": 5, "mysql": 4}
    
    -- 绩效指标
    total_resolved  INTEGER DEFAULT 0,
    avg_resolution_hours DECIMAL(6,2),
    satisfaction_score DECIMAL(3,2) DEFAULT 5.00,
    
    -- 当前负载
    current_load    INTEGER DEFAULT 0,
    max_load        INTEGER DEFAULT 10,
    
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 6.1.4 知识库表 (knowledge_base)

```sql
CREATE TABLE knowledge_base (
    id              BIGSERIAL PRIMARY KEY,
    title           VARCHAR(500) NOT NULL,
    content         TEXT NOT NULL,
    
    -- 向量嵌入 (pgvector)
    embedding       VECTOR(1536),
    
    category        VARCHAR(50),
    source_type     VARCHAR(50),  -- 'ticket', 'manual', 'document'
    source_id       BIGINT,
    
    -- 质量指标
    view_count      INTEGER DEFAULT 0,
    helpful_count   INTEGER DEFAULT 0,
    not_helpful_count INTEGER DEFAULT 0,
    
    status          VARCHAR(20) DEFAULT 'draft',  -- draft, published, archived
    
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_category (category),
    INDEX idx_status (status)
);

-- 启用向量相似度搜索
CREATE INDEX ON knowledge_base USING ivfflat (embedding vector_cosine_ops);
```

#### 6.1.5 操作日志表 (operation_log)

```sql
CREATE TABLE operation_log (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT,
    user_name       VARCHAR(100),
    action          VARCHAR(50) NOT NULL,
    resource_type   VARCHAR(50),
    resource_id     BIGINT,
    details         JSONB,
    ip_address      VARCHAR(50),
    user_agent      VARCHAR(500),
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
